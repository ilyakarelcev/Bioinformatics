По идее основный источник данных это Basic Local Alignment Search Tool (BLAST). Конкретно база данных на которой построен BLAST называется GenBank, но думаю лучше использовать её через BLAST. Дополнительные информации о протеинах можем брать с UniProt. Также есть ещё куча других баз данных, типо BioSamples, ENA, но для начала начнём с минимальным набором.

def translate_dna_to_protein(dna_sequence):
if len(dna_sequence) % 3 != 0:
raise Exception("The coding sequence (CDS) length must be divisible by 3, "
"as each codon is 3 nucleotides long")

def codons(sequence):
return (sequence[i:i+3] for i in range(0, len(sequence), 3))

def translate_codon(codon):
ambiguous_bases = {'N', 'R', 'Y', 'K', 'M', 'S', 'W', 'B', 'D', 'H', 'V'}
if any(base in ambiguous_bases for base in codon):
return 'X'
return standard_codon_table.get(codon, 'X')

def build_protein(codons_iter):
protein = []
for codon in codons_iter:
amino_acid = translate_codon(codon)
if amino_acid == '*':
break
protein.append(amino_acid)
return ''.join(protein)

upper_dna = dna_sequence.upper()
return build_protein(codons(upper_dna))

Помимо обычного ATCG, могут встречаться символы, обозначающие неоднозначные нуклеотиды. Это может быть связано с некачественным секвенированием, деградацией образцов и т.д. Это похоже на шум или неопределенные значения, как и во всех реальных данных. Когда мы встречаем кодон, содержащий неоднозначный нуклеотид, мы обычно транслируем его как X для любой аминокислоты.
19:44
Классификация последовательности на основе самой последовательности сложна. Буквы ATCG могут быть нуклеотидами или аминокислотами, хотя в одном или другом случае обозначают совершенно разные вещи. Из 4 нуклеотидов и 20 аминокислот следует, что в белках встречаются буквы вне ATCG. Белковая последовательность не обязана делиться на 3. Но любую ДНК последовательность можно ошибочно принять за белок. Если обратим внимание, неоднозначные нуклеотиды RYKMSWDHV встречаются только в ДНК, а ZJUOX — только в белках, но они в принципе редко встречаются. Стоп-кодоны (TAA, TAG, TGA) обозначают конец и помогают классификации, но пользователь может вводить неполные последовательности.

Всем привет, поднял графовую БД
Загрузил обогащенные данные и накидал простого агента который общается с этой бд

Пример ответа
You> For accession A2ACJ2, summarize common diseases across its neighbors and give a short interpretation. Ответь на русском

Agent> Для белка с аксесcионом A2ACJ2 среди его соседей по графу обнаружены общие заболевания, связанные с синдромом Германски-Пудлака-подобным синдромом. Этот синдром характеризуется гипопигментацией глаз и шерсти, значительным уменьшением количества и морфологическими изменениями меланосом, нарушением секреции лизосомальных ферментов в почечных канальцах, а также функциональными нарушениями тромбоцитов, что приводит к удлиненному времени кровотечения.

Примеры соседних белков, связанных с этим заболеванием, включают Hps6 и Hps5, которые являются членами комплекса BLOC-2 и играют ключевую роль в развитии данного синдрома.

Интерпретация: белок A2ACJ2, судя по его близости к белкам, ассоциированным с синдромом Германски-Пудлака, может быть вовлечен в биологические процессы, связанные с пигментацией, функцией лизосом и тромбоцитов. So, I’ve got reranking hammered out using gemicli. I took a similar approach to the one Ilya used, though with a few tweaks. First, I built an HNSW index with FAISS and used that to rank by protein sequence. Then, for the reranking step, I ran a brute-force k-NN search—again with FAISS—but this time incorporating context. For the initial ranking, I used the same T5 model they did to embed the sequences from the database. For the smaller reranking step, I went with the simple model we used for the second homework. I might experiment with ColBERT or something more powerful later, but for now, this seems to do the job.
